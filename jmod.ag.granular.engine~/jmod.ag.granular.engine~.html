ola/interpolate decimal 0.000000 1.000000 both none none none none 0 toggles sub sample accurate grain triggering. Otherwise it doesn't give any audible benefits and unnecessarily uses CPU resources (0: off, 1: on)

Example: source = 3.6 -> source1 = 3, source2 = 4, interpolation factor = 0.6
